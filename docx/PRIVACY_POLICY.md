1. Общие положения

1.1. PhishGuard — это браузерное расширение и серверная часть (API), предназначенные для анализа URL адресов и текстового контента на признаки фишинга.

1.2. Настоящая Политика конфиденциальности (далее — «Политика») описывает, какие данные обрабатываются при использовании PhishGuard, как и для чего они используются.

1.3. Используя PhishGuard, вы соглашаетесь с условиями данной Политики.

2. Какие данные обрабатываются и с какой целью

PhishGuard обрабатывает только те данные, которые необходимы для выполнения анализа:

2.1. URL адрес для анализа
Когда вы инициируете проверку (через контекстное меню или подсказку), URL адрес передаётся на сервер PhishGuard.  
Цель: анализ домена и структуры URL, проверка редиректов, TLS/HTTPS, а также (при включённом анализе контента) загрузка страницы для анализа HTML.

2.2. Текстовый контент (опционально)  
Если вы используете функцию «Проверить выделенный текст», выделенный фрагмент может быть передан на сервер.  
Цель: анализ текста на наличие фраз и триггеров, характерных для социальной инженерии (срочность, угрозы, финансовые приманки и т.п.).

2.3. Почтовые заголовки (опционально) 
Если вы передаёте заголовки письма, они могут быть обработаны сервером.  
Цель: анализ SPF/DKIM/DMARC и несоответствий From/Reply To/Return Path.

2.4. Технические данные соединения 
При обращении к серверу неизбежно могут обрабатываться технические данные (например, IP адрес, дата/время запроса, технические параметры HTTP соединения).  
Цель: обеспечение работы сервиса, защита от злоупотреблений и диагностика ошибок.

3. Как и где происходит обработка данных

3.1. Процесс работы:  
1) Расширение отправляет HTTPS запрос на сервер PhishGuard, содержащий URL и/или текст.  
2) Сервер выполняет многоуровневый анализ в режиме реального времени.  
3) Сервер возвращает результат (отчёт) в формате JSON обратно в расширение.

3.2. Публичный адрес сервера (API): https://phishguard-a43g.onrender.com

3.3. Сервер размещён на платформе Render.com. Входящие HTTPS запросы принимаются инфраструктурой Render и далее перенаправляются в веб сервис (TLS termination на стороне Render).  
Справка Render: https://render.com/docs/web-services#port-binding

3.4. Регион размещения сервиса (если используется): Frankfurt (Central EU).

4. Хранение данных

4.1. PhishGuard не сохраняет содержимое анализируемых данных (URL, текст, заголовки) в собственной базе данных и не ведёт историю проверок как часть логики приложения.

4.2. При этом возможны технические логи на стороне хостинг провайдера (Render) и сетевой инфраструктуры (например, для мониторинга, безопасности и отладки). Сроки хранения таких журналов зависят от политики и настроек Render.  
Политика Render: https://render.com/privacy

5. Передача данных третьим сторонам

5.1. Сторонние базы угроз (опционально) 
В процессе анализа проверяемый URL может быть отправлен во внешние сервисы:  
-Google Safe Browsing (Google LLC)— проверка URL по базе небезопасных ресурсов;  
-URLHaus (abuse.ch) — сверка URL с базой известных вредоносных ссылок.  

Передаётся только URL, необходимый для проверки.

5.2. Загрузка анализируемой страницы 
Если включён анализ HTML контента, сервер выполняет запрос к проверяемому сайту для получения HTML страницы. В этом случае проверяемый сайт может получить технические данные запроса (например, IP адрес сервера и User Agent) и факт обращения к URL.

5.3. WHOIS/DNS  
Для анализа домена могут выполняться запросы к WHOIS источникам и/или сетевым сервисам. Передаётся доменное имя.

6.2. Вы можете удалить расширение из браузера — после этого расширение перестанет отправлять запросы на сервер.

7.2. Мы не используем данные для профилирования пользователей и не формируем пользовательские профили на основе запросов.
